# Contributing to Agent Character Design

Thanks for your interest. This is a young discipline — the early shape matters a lot, and contributions of every size are welcome.

## Types of contribution

### 1. Counter-examples (highest value)
Found a claim in the Field Guide that doesn't hold in your system? **Open an issue** with the specific claim + your evidence. Falsification is how the discipline matures.

### 2. Case studies
Have a Level 2 pattern you use in production? PR a case study to `docs/case-studies/`. Format: 1-page minimum, concrete system + problem + mechanism + result.

### 3. Chapter contributions
Chapters 3-6 are open. If you want to draft one, open an issue first to discuss scope and approach.

### 4. Tooling
Tools live in separate repos under [agent-character-design](https://github.com/agent-character-design). Reach out on an issue before starting — some tools are already in design.

### 5. Terminology
ACD is building a vocabulary. If a term feels off or a new one needed, open an issue. Naming is hard and worth discussing.

## What we're NOT looking for

- "This should be renamed to X" without a concrete reason
- Adding more trait labels without a multi-track justification
- Level 1 techniques dressed up as Level 2
- Purely theoretical contributions with no production evidence

## Process

1. Fork the repo
2. Create a branch: `git checkout -b your-topic`
3. Make changes
4. Open a PR with a clear description + any linked issues

For chapter PRs, please include:
- The specific claim being added/changed
- Evidence (academic paper, production log, measurable result)
- Counter-argument (steelman the opposite view, then explain why the claim still holds)

## Code of Conduct

Be rigorous and kind. Challenge ideas, not people. If you notice bad behavior, contact zhouzico@gmail.com.
